OSLF and Spatial-Behavioral Types: Querying Code by Behavior

“Turning type checking into a query engine for code.” — Lucius Meredith (Jun 2025)

In brief: Spatial-behavioral type systems can support compile-time analysis for whole classes of safety errors. Turned sideways, a type checker becomes a query engine for code — searching programs by behavior rather than by labels. OSLF: algorithms that take an operational semantics and generate a spatial-behavioral type system for classification and search.

Key takeaways

  • Spatial-behavioral type systems can support compile-time analysis for whole classes of safety errors.
  • Turned sideways, a type checker becomes a query engine for code — searching programs by behavior rather than by labels.
  • OSLF: algorithms that take an operational semantics and generate a spatial-behavioral type system for classification and search.

Key questions

  • How does OSLF-based search scale to large codebases?
  • What classes of bugs does spatial-behavioral typing catch that conventional types miss?
  • How close is this to production-ready tooling?

Why it matters for developers

This reframes ‘finding the right code’ and ‘preventing catastrophic bugs’ as one problem: classification. Type checking becomes both a safety tool and a search tool.
